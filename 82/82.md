```python
class Solution:
    def deleteDuplicates(self, head: Optional[ListNode]) -> Optional[ListNode]:
        dummy = ListNode(None, None)

        duplicated = False
        last_unique = dummy
        node = head
        
        while node:
            if node.next and node.val == node.next.val:
                duplicated = True
            else:     
                if not duplicated:
                    last_unique.next = node
                    last_unique = node
                duplicated = False
            node = node.next
            
        last_unique.next = None
    
        return dummy.next
```

```python
class Solution:
    def deleteDuplicates(self, head: Optional[ListNode]) -> Optional[ListNode]:
        dummy = ListNode(0, head)   # 番兵
        prev  = dummy               # 重複がないことが確定した最後尾
        curr  = head                # 現在走査中のノード

        while curr:
            # curr が重複列の先頭かどうかをチェック
            duplicated = False
            while curr.next and curr.val == curr.next.val:
                curr = curr.next    # 同じ値の末尾まで進む
                duplicated = True

            if duplicated:
                # curr は重複列の末尾。重複ごとスキップ
                prev.next = curr.next
            else:
                # 重複していなければ prev を 1 ノード進める
                prev = prev.next

            curr = curr.next        # 次の候補へ

        return dummy.next
```

いろいろやり方があって面白い。
duplicatedなどのフラグは、while文の中で使う分には悪くないが、自分のstep1の実装のように外で使ってしまうと結構見にくくなってしまうなという印象。

「次にvalueが変わるノードを探す」という操作を関数に切り出して使っている回答もあって面白かった。
ただ、このようにする場合は関数が何をしているのか（何を返すのか）を中身を見ないでも明確に理解できるような名前を付けることができないと、
結局その関数の中身の実装を確認することになってしまって、関数化せずにそのまま書いた場合とそこまで読みやすさは変わらないかもという感じがした。
（あるいは、関数の名前だけでの説明が難しそうな場合は、その関数の冒頭にコメントをつけておくというやり方でもよいかも）

あと、最初の自分の実装は「nodeがあるときに、それがuniqueかどうか確認し、もしそうだったらlast_uniqueとつなげる」という形で書いており、「uniqueでない場合はnextにとりあえずつなげておく」というのは書いていないので、
`while node`のような条件で回している限り、最後にlast_unique.next = Noneにあたる処理を追加しなければいけないのがちょっといやだなという感じがある。（ [1, 3, 3, 3] のような最後に同じ数字が連続で入っている入力に対応するため）

## ステップ3: 最終コード（エラーを出さずに3回書いてみる）

```python
class Solution:
    def deleteDuplicates(self, head: Optional[ListNode]) -> Optional[ListNode]:
        dummy = ListNode(None, head)

        last_unique = dummy
        node = head

        while node:
            if not node.next or node.val != node.next.val:
                last_unique = node
                node = node.next
            else:
                val = node.val
                while node and node.val == val:
                    node = node.next
                last_unique.next = node
                
        new_head = dummy.next
        return new_head
```

```python
class Solution:
    def deleteDuplicates(self, head: Optional[ListNode]) -> Optional[ListNode]:
        dummy = ListNode(None, head)

        last_unique = dummy
        node = head

        while node:
            if node.next and node.val == node.next.val:
                val = node.val
                while node and node.val == val:
                    node = node.next
                last_unique.next = node
            else:
                last_unique = node
                node = node.next
                
        new_head = dummy.next
        return new_head
```
