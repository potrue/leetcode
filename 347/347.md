## 何も見ずに解いてみる

```python
class Solution:
    def topKFrequent(self, nums: List[int], k: int) -> List[int]:
        num_to_appearance = defaultdict(int)
        for num in nums:
            num_to_appearance[num] += 1
        num_to_appearance_sorted = dict(sorted(num_to_appearance.items(), key=lambda item: item[1], reverse=True))
        result = list(num_to_appearance_sorted.keys())[:k]
        return result
```

Follow up: Your algorithm's time complexity must be better than O(n log n), where n is the array's size.
上の回答は恐らくsortの時にO(nlogn)かかっているので満たさないはず、どうやるんだろう？

Counterのmost_common関数はheapq.nlargestを用いていて、heapq.nlargestはheappushを使ったやり方とほぼ同じ方法で結果を計算しているっぽい（ただしこれは最後ソートされた状態で結果が返ってくる）。

BucketSortという方法もあるらしい。これはそれぞれのnumが出現する回数から、そのようなnumを抽出する逆引き的なリストを作り、最後に頻度が大きい順にそのリストを検索するという方法。

辞書を作るときに逆からの辞書を同時に作成しておくようなデータ構造（bidictというみたい）がもともと作ってあればもう少し簡潔に書けるのかもしれない。

## 最終コード

シンプルな方法

```python
class Solution:
    def topKFrequent(self, nums: List[int], k: int) -> List[int]:
        counter = Counter(nums)
        result = [num for num, count in counter.most_common(k)]
        return result
```

BucketSort（ただし上のコードのほうが実行時間でいえば早かったです）

```python
class Solution:
    def topKFrequent(self, nums: List[int], k: int) -> List[int]:
        counter = Counter(nums)
        bucket = [list() for _ in range(len(nums) + 1)]
        for num, count in counter.items():
            bucket[count].append(num)
        result = []
        for count in range(len(nums), 0, -1):
            result.extend(bucket[count])
            if len(result) == k:
                break
        return result
```
