```python
class KthLargest:

    def __init__(self, k: int, nums: List[int]):
        nums.sort(reverse=True)
        self.k = k
        self.k_largest_nums = nums[:k]
        if len(self.k_largest_nums) >= k:
            self.cutoff = self.k_largest_nums[k-1]
        else:
            self.cutoff = None

    def add(self, val: int) -> int:
        if self.cutoff is None or val > self.cutoff:
            bisect.insort(self.k_largest_nums, val, key=lambda x: -x)
            if len(self.k_largest_nums) > self.k:
                self.k_largest_nums.pop()
        self.cutoff = self.k_largest_nums[-1]
        return self.cutoff
```

heapqの実装はwhileではなく再帰を使った実装も結構見る。どっちがわかりやすいだろう？
Heap, PriorityQueueの問題群を解き終わったら何かしらのPriority Queueの実装を別のPRで作ってみようと思います。

そういえばこの問題、スコアを新しく追加する関数が必ずintを返す設定になっていて、今までに登録されているスコアがk個以下の場合はNoneを返してください、みたいな指定が無いの結構違和感があります。

初期化の部分は、`for num in nums:`で追加していくやり方と、`heapq.heapify(nums)`でいったん全部変換してからk個以下になるまでheappopしていくやり方を思いつきましたが、
kとnumsの長さの組み合わせ（例えば2, 10000とか）によってはだいぶ処理が余計にかかっちゃいそうなのでfor文で回す方法にしてみました。（すべて保存しておく場合はheapifyのほうが若干はやいみたいですが）

## 最終コード（エラーを出さずに3回書いてみる）

class KthLargest:

    def __init__(self, k: int, nums: List[int]):
        self._k = k
        self._top_k_heap = []
        for num in nums:
            self.add(num)

    def add(self, val: int) -> int:
        if len(self._top_k_heap) < self._k:
            heapq.heappush(self._top_k_heap, val)
        else:
            heapq.heappushpop(self._top_k_heap, val)
        return self._top_k_heap[0]
