## 何も見ずに解いてみる

```cpp
class Solution {
public:
    int maxSubArray(vector<int>& nums) {
        vector<int> cumsums = {0};
        int min_cumsum = 0;
        int max_cumsum_diff = numeric_limits<int>::min();
        for (int i = 0; i < nums.size(); i++) {
            int new_cumsum = cumsums.back() + nums[i];
            cumsums.push_back(new_cumsum);
            max_cumsum_diff = max(max_cumsum_diff, new_cumsum - min_cumsum);
            min_cumsum = min(min_cumsum, new_cumsum);
        }
        return max_cumsum_diff;
    }
};
```

## いろいろ調べてみる

自分が最初に書いたコード、cumsumを配列として保存する必要ないですね。int cumsumを一つもって逐一更新していけばよさそうです。

今まで出てきた最小の累積和を保存する方法と、そのindexを終点とする中で合計が最大の部分列の合計を保存する方法を比較すると、実質的に`new_cumsum - min_cumsum`がそのindexを終点とする中で合計が最大の部分列の合計になるという点で、裏返しになっているかも。  
min_cumsumの最小値が更新されれば、その次のタイミングで「直前のindexを終点とする中で合計が最大の部分列の合計よりもそのindex単体のほうが大きい」という状態になる

Divide and Conquer、これを端からやっていくと、例えば右端からそれぞれのindexについて、左側の部分列についてはその関数自信の再帰に任せつつ、  
今のindexを部分列に入れる場合の最大値を探すっていうのをやる感じになって、イメージO(n^2)の累積和の差をすべて調べる解法と近いような感じになるかも。  
あと、配列全体を半分ずつに分割していくトップダウン的な方法ではなく、配列一つ一つの要素がばらばらになった状態で初めてから、ボトムアップ的に最大値を構築できないかな？  
(ある部分配列の左端のindex, ある部分配列の右端のindex, その部分配列の中で与えられる最大値) みたいな組が並んでいる配列を作って、  
二つの部分配列がつながっているとして、左の部分配列の最大値、右の部分配列の最大値がわかっているとすれば、  

- 左の部分配列の最大値  
- 右の部分配列の最大値  
- 左の部分配列の右端を使う場合の左の部分配列の中の最大値 + 右の部分配列の左端を使う場合の右の部分配列の中の最大値

の3つが候補となるので、maxをとってつながった部分配列の最大値として次のラウンドでiterateするための配列に入れておく、みたいなことをすればできそう（またの機会に書いてみます）

## 最終コード

最大値とか最小値を更新するとき、`min(min_cumsum_so_far, cumsum)`みたいにもともとの値を最初に持ってくるのと、`min(cumsum, min_cumsum_so_far)`と後に持ってくるのだとどっちが見やすいですかね？

```cpp
class Solution {
public:
    int maxSubArray(vector<int>& nums) {
        int cumsum = 0;
        int min_cumsum_so_far = 0;
        int max_cumsum_diff = numeric_limits<int>::min();
        for (int num : nums) {
            cumsum += num;
            max_cumsum_diff = max(max_cumsum_diff, cumsum - min_cumsum_so_far);
            min_cumsum_so_far = min(min_cumsum_so_far, cumsum);
        }
        return max_cumsum_diff;
    }
};
```
