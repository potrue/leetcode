## 何も見ずに解いてみる

```python
class Solution:
    def firstUniqChar(self, s: str) -> int:
        char_to_count = Counter(s)
        for char in char_to_count:
            if char_to_count[char] == 1:
                return s.find(char)
        return -1
```

最初のコードの方がleetcodeの実行時間的には早いは早かったが、こちらの方が実装としては自然か。

```python
class Solution:
    def firstUniqChar(self, s: str) -> int:
        char_to_count = Counter(s)
        for index, char in enumerate(s):
            if char_to_count[char] == 1:
                return index
        return -1
```

indexを保存しておいて、辞書の順番を利用しつつもindexを探すためにfindを回す必要がないようにしても良さそう。

```python
class Solution:
    def firstUniqChar(self, s: str) -> int:
        char_to_indices = defaultdict(list)
        for index, char in enumerate(s):
            char_to_indices[char].append(index)
        for char, indices in char_to_indices.items():
            if len(indices) == 1:
                return indices[0]
        return -1
```

登場する文字は少ないけど文字列の長さがすごく長いので同じ文字が何回も登場するというような場合には、辞書のキーを順番にイテレートしていくほうが良いと思われる。
