```python
class Solution:
    def levelOrder(self, root: Optional[TreeNode]) -> List[List[int]]:
        if not root:
            return []
        result = []
        queue = [(root, 0)]
        while queue:
            node, depth = queue.pop()
            if depth >= len(result):
                result.append([])
            result[depth].append(node.val)
            if node.right:
                queue.append((node.right, depth + 1))
            if node.left:
                queue.append((node.left, depth + 1))
        return result
```

これ自体を再帰の関数として使うこともできそう

```python
class Solution:
    def levelOrder(self, root: Optional[TreeNode]) -> List[List[int]]:
        if not root:
            return []
        left_level_order = self.levelOrder(root.left)
        right_level_order = self.levelOrder(root.right)
        max_depth = max(len(left_level_order), len(right_level_order)) + 1
        root_level_order = [[] for _ in range(max_depth)]
        root_level_order[0].append(root.val)
        for depth, nodes in enumerate(left_level_order):
            root_level_order[depth + 1].extend(nodes)
        for depth, nodes in enumerate(right_level_order):
            root_level_order[depth + 1].extend(nodes)
        return root_level_order
```

しかし、この方法だとそれぞれのノードについてそのノードが入ったリストはrootにとっての深さ分の回数構築されることになるので、結構計算量が増えて遅くなるかも  
ただ、偏りのない(leftとrightがバランスよく埋まっている)木なら、rootの持つノードの数をnとして、O(n log n)で済む？（違ったらすみません）

## 最終コード

nodeがNoneでも一旦入れてしまう感じにしてみました

```python
class Solution:
    def levelOrder(self, root: Optional[TreeNode]) -> List[List[int]]:
        result = []
        queue = deque()
        queue.append((root, 0))
        while queue:
            node, depth = queue.popleft()
            if not node:
                continue
            if depth >= len(result):
                result.append([])
            result[depth].append(node.val)
            queue.append((node.left, depth + 1))
            queue.append((node.right, depth + 1))
        return result
```
