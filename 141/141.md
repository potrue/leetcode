## ステップ1: 何も見ずに解いてみる

```python

# Definition for singly-linked list.
# class ListNode(object):
#     def __init__(self, x):
#         self.val = x
#         self.next = None

class Solution(object):
    def hasCycle(self, head):
        """
        :type head: ListNode
        :rtype: bool
        """

        visited = set()
        current = head

        if not current:
            return False
            
        while current.next:
            visited.add(current)
            current = current.next
            if current in visited:
                return True
                
        return False
```

## ステップ2: いろいろ調べる

GPT先生に聞いたところ
Input: head = [3,2,0,-4], pos = 1
とかって出てきてるのは内部実行用の情報であって、実際に関数の入力として渡されるのは一番先頭のListNode(3)っぽい

関数の入力という意味でInputという単語が使われているわけではないらしい

自分の元の実装はNをリストの長さとすると
時間計算量O(N)
メモリO(N)

メモリO(N)で解く方法として、進む長さが違うポインタを用意して一致したらサイクルがあると判定するというアルゴリズムがあるらしい（GPT出力コード）

```python
def hasCycle(head: Optional[ListNode]) -> bool:
    slow = fast = head
    while fast and fast.next:
        slow = slow.next        # 1 つ進む
        fast = fast.next.next   # 2 つ進む
        if slow is fast:        # どこかで追いつけばサイクル
            return True
    return False
```

上のコード、fast.next.nextがなかったときにエラーが出るんじゃないか？と思ったけど、self.next = NoneがListNodeのinitに定義されているから大丈夫ということか

objectの機能なども軽く見てみましたが深入りすると大変そう、pythonのクラスが多重継承できることも初めて知りました

## ステップ3: 最終コード（エラーを出さずに3回書いてみる）

```python
class Solution:
    def hasCycle(self, head):
        slow = head
        fast = head
        while fast and fast.next:
            slow = slow.next
            fast = fast.next.next
            if slow is fast:
                return True
        return False
```
